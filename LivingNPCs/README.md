# LivingNPCs

LivingNPCs 是一个星露谷物语 SMAPI companion mod，目标是给 NPC 增加一层更接近真实生活的：

- 状态。
- 记忆。
- 关系节奏。
- 小行为。
- 对 AI 对话结果的受控反馈。

它不是 ValleyTalk 的替代品，而是它的“行为与连续性层”：

- **ValleyTalk** 负责生成 AI 对话。
- **LivingNPCs** 负责让 NPC 记住过去、带着关系继续说话，并在安全范围内做出一点实际反应。

更完整的项目说明见仓库根目录 [README.md](../README.md)。

## 当前能力

### 行为层

已实现的微行为：

- `FacePlayer`：转向玩家。
- `Emote`：转向玩家并显示表情。
- `ApproachPlayer`：走近玩家。
- `Pause`：停下并注意玩家。
- `LookAround`：环顾四周。
- `StepAway`：在合适时后退一步。

行为可由两种方式触发：

- 玩家按快捷键手动测试。
- NPC 在附近时按概率被动触发。

行为选择会参考：

- 当前关系层级。
- 长期熟悉度。
- 今天已经聊了几次。
- 当前地点、时间、天气、附近 NPC。
- 角色自己的轻量性格倾向。

### 记忆层

LivingNPCs 目前保存：

- 最近行为记录。
- 对话开始记录。
- 送礼记录及可推断的喜好层级。
- 事件 / 节日互动记录。
- NPC 当前状态：
  - 心情。
  - 注意度。
  - 开放度。
  - 当前回应倾向。
- NPC 对玩家的长期熟悉度。
- 每日互动节奏和关系舒适区。
- AI 提炼出的长期记忆：
  - 事实。
  - 偏好。
  - 承诺。
  - 边界。
  - 关系信息。
- 单独的玩家偏好记忆：
  - 喜欢的物品类别。
  - 讨厌的物品。
  - 生活习惯。
  - 价值观和态度。
  - 目标和计划。
- 单独的长期约定：
  - 再见面。
  - 一起去某处。
  - 帮忙做某事。
  - 待履行 / 已履行 / 已过期状态。
- 单独的人际情绪和冲突记忆：
  - 当前情绪。
  - 情绪强度。
  - 冲突原因与严重度。
  - 恢复状态。
- 玩家专属称呼记忆。
- NPC 已执行过的 AI 世界动作。

每天开始时：

- 短期状态会向普通值回落。
- 人际情绪会向平静回落。
- 未解决冲突会按配置自然缓和。
- 当日计数会重置。
- 长期熟悉度和重要长期记忆不会像短期心情那样被清空。

### 情绪与冲突

当前长期情绪分为：

- `Happy`
- `Calm`
- `Uneasy`
- `Upset`
- `Angry`
- `Sad`

触发来源包括：

- 玩家对话里真实造成的情绪变化。
- 玩家道歉或修复关系。
- 送礼质量。

冲突不是普通尴尬，而是会继续影响关系的事件，例如明显冒犯、越界、失约等。LivingNPCs 会记录：

- 冲突原因。
- 严重度。
- 当前状态：`Active`、`Recovering`、`Resolved`
- 修复分数、道歉次数、解决时间。

当前恢复方式：

- AI 判断玩家确实道歉。
- 送出喜欢 / 最爱的礼物。
- 时间自然流逝。

未解决冲突会影响行为和对话：

- NPC 更少靠近玩家。
- 更容易保留距离。
- 严重冲突下会阻止送礼、送钱、帮忙等亲近动作。
- ValleyTalk 会在生成对白时收到这层状态，因此 NPC 可以更冷淡，或在刚和解后自然说出“之前的事已经过去了”。

### 关系节奏

LivingNPCs 会同时看：

- 原版好感心数。
- LivingNPCs 长期熟悉度。
- 连续聊天天数。
- 今天第几次聊天。
- 距离上次聊天隔了多久。

据此区分：

- `Distant`
- `Familiar`
- `Friendly`
- `Trusted`
- `Intimate`

不同关系层级有不同的“每天聊几次还算自然”的舒适上限。这样 NPC 不会因为接入 AI 后就一直机械热情。

### ValleyTalk 集成

LivingNPCs 会把一份隐藏上下文推给 ValleyTalk，里面包含：

- 当前关系节奏。
- 当前场景。
- 当前 mood / inclination。
- 当前长期情绪与冲突。
- 最近送礼和事件。
- 长期记忆。
- 玩家偏好记忆。
- 待履行或失约的长期约定。
- 玩家称呼偏好。
- 最近几条关键互动。

这份上下文只用于让 AI 更自然地延续关系，不应该被 NPC 原样说出来。

### AI 对话结果回写

ValleyTalk 当前会把以下隐藏元数据回传给 LivingNPCs：

- `rapportDelta`：本轮 AI 对话额外好感。
- `memories`：值得保留的长期记忆。
- `endConversation`：是否自然结束这轮交流。
- `ambientFollowUp`：稍后显示的头顶短句。
- `emotionImpact`：这轮交流造成的人际情绪变化，以及是否构成道歉 / 修复。
- `conflicts`：需要长期记住的冲突事件。
- `actions`：最多一个受控世界动作请求。

当前额外好感规则：

- AI 自己判断这轮聊得怎么样。
- 单次范围 `0-30`。
- 每个 NPC 每天总额外上限默认 `30`。
- 这是在原版每日对话好感之外追加的奖励。

### AI 世界动作

LivingNPCs 当前允许的世界动作只有：

- `give_small_gift`
- `give_meaningful_gift`
- `give_money`
- `water_nearby_crops`
- `walk_together`
- `escort_to_location`
- `festival_interaction`
- `assist_quest`

但 AI 只能“申请”，不能直接执行。LivingNPCs 会重新检查：

- 关系是否足够。
- 当前位置是否合适。
- 今天是否已经执行过。
- 是否在冷却中。
- 是否超过配置上限。

当前主要规则：

- 小礼物：至少有一点熟悉；每天每个 NPC 最多一次 AI 礼物。
- 有意义礼物：至少友好，并且满足高关系 / 近期事件 / 重要长期记忆之一；默认 7 天冷却。
- 送钱：至少友好；金额默认 100g，受单次上限控制。
- 浇水：至少友好；只能在农场；只影响附近已种植但未浇水的作物。
- 同行：至少有一点熟悉；临时跟随，不改永久日程。
- 陪去地点：只做有时限、路径受限的短时陪走，不做跨地图瞬移或永久日程修改。
- 节日互动：只在事件 / 节日场景中允许。
- 任务协助：只围绕已有任务做轻量协助，不直接完成任务。

### 礼物选择

礼物分两层：

#### 小礼物

偏向日常、低价值，当前会结合：

- NPC 性格标签。
- 好感和熟悉度。
- 季节。
- 职业 / 背景。
- 当前对话内容。
- 长期记忆里出现的主题。
- 玩家偏好记忆；这部分权重高于普通主题记忆。

#### 有意义的礼物

偏向关系更深或特殊节点，只有在条件更强时才可能出现。它不是“小礼物升级版”，而是更少、更郑重的一层。

如果 NPC 已经记得玩家喜欢某类物品，礼物选择会明显向该类倾斜；如果记得玩家讨厌某类物品，相关候选会被压低。ValleyTalk 也会在合适时自然提一句类似“记得你喜欢这个”，让礼物和记忆接上，而不是凭空出现。

### 长期约定

当双方在 AI 对话里明确约好了未来再见、一起去某处、之后帮忙做某事、一起庆祝，或一起进行某项活动时，ValleyTalk 会返回一条结构化约定，LivingNPCs 会保存：

- 类型：`meet_again`、`go_together`、`help_task`、`celebrate_together`、`share_activity`
- 约定时间。
- 约定地点。
- 当前状态。

到期后：

- 如果玩家和 NPC 都在约定地点，NPC 会主动靠近并打招呼。
- 约定前一天，如果再次对话，NPC 会自然提醒一次第二天的安排。
- 约定当天清晨，如果双方在附近，NPC 有概率主动用头顶气泡提醒。
- 如果 NPC 到了而玩家没到，它会进入“等待约定”的上下文。
- 如果超过宽限时间仍未完成，约定会变成 `Expired`。
- 下次正常对话开始时，ValleyTalk 会优先知道这次失约，并自然提起。
- 如果约定被完成，NPC 会获得一点关系反馈，并写下一条单独的“共同经历”。
- 几天后，NPC 可以主动提起这次共同经历；愉快的共同经历也会让它更容易提出相似的新约定。
- 失约会降低“履约信任”，连续失约会造成更明显的失望和 `promise` 冲突；玩家若认真道歉并重新给出一个具体约定，可以开始修复这层信任。

当前版本不改 NPC 永久日程，也不跨地图瞬移；这是故意留下的安全边界。

### 自定义 NPC 兼容

- 原版角色有内置倾向。
- SVE 当前内置识别到的角色已经补齐手写摘要，并处理了常见内部别名。
- RSV 已补充一批主要村民的手写摘要；其余角色仍可依靠 `Data/Characters` 做基础推断。
- 对未手写配置的 NPC，会从 `Data/Characters` 推断基础性格。
- 已对 SVE 和 RSV 做了专门识别：
  - 有些角色有手写摘要。
  - 其余角色会保留所属 Mod 的世界观提示，并回退到基础推断。
- `npc_profiles/` 目录支持社区直接放入 JSON 人物资料：
  - 可新增自定义 NPC；
  - 也可覆盖已有条目做修正；
  - 模板与提交流程见 [`npc_profiles/README.md`](./npc_profiles/README.md)。

## 使用方法

默认快捷键：

- `LeftShift + H`：触发附近 NPC 的一次小行为。
- `LeftShift + J`：查看附近 NPC 的状态和记忆。

Generic Mod Config Menu 中可以调：

- 行为快捷键。
- 记忆数量。
- AI 对话额外好感。
- 世界动作开关。
- 礼物、送钱、浇水、同行的限制。
- 每日情绪衰减、每日冲突衰减。
- 被动行为概率。
- ValleyTalk 桥接。
- AI 行为规划器。

## AI 行为规划器

这是一个与 ValleyTalk 对话生成分开的可选功能，默认关闭。

启用后，LivingNPCs 可以向 OpenAI-compatible 接口请求一个行为意图：

```json
{
  "intent": "FacePlayer",
  "reason": "short in-world reason",
  "emoteId": 16
}
```

但即使模型返回了内容，LivingNPCs 仍会做本地安全校验；失败时会自动回退到规则行为，不会因为接口异常卡住游戏。

## 开发构建

```powershell
dotnet build
```

如果已安装 `Pathoschild.Stardew.ModBuildConfig`，构建成功后会自动部署到游戏的 `Mods` 目录。
