# LivingNPCs

LivingNPCs 是一个星露谷物语 SMAPI companion mod，目标是给 NPC 增加一层更接近真实生活的：

- 状态。
- 记忆。
- 关系节奏。
- 小行为。
- 对 AI 对话结果的受控反馈。

它不是 ValleyTalk 的替代品，而是它的“行为与连续性层”：

- **ValleyTalk** 负责生成 AI 对话。
- **LivingNPCs** 负责让 NPC 记住过去、带着关系继续说话，并在安全范围内做出一点实际反应。

更完整的项目说明见仓库根目录 [README.md](../README.md)。

## 当前能力

### 行为层

已实现的微行为：

- `FacePlayer`：转向玩家。
- `Emote`：转向玩家并显示表情。
- `ApproachPlayer`：走近玩家。
- `Pause`：停下并注意玩家。
- `LookAround`：环顾四周。
- `StepAway`：在合适时后退一步。

另外还有一层 **对话驱动行为**：

- `visit_location`：NPC 在对话里真心提到想去某处，之后若来到那个地点，会把这件事接上。
- `comforted` / `stay_near` / `pause_to_talk`：玩家安慰或拉近关系后，NPC 会更愿意靠近、停留、继续回应。
- `offended` / `give_space`：被冒犯或需要空间时，NPC 后续行为会更疏远。

行为可由两种方式触发：

- 玩家按快捷键手动测试。
- NPC 在附近时按概率被动触发。

行为选择会参考：

- 当前关系层级。
- 长期熟悉度。
- 今天已经聊了几次。
- 当前地点、时间、天气、附近 NPC。
- 角色自己的轻量性格倾向。

### 记忆层

LivingNPCs 目前保存：

- 最近行为记录。
- 对话开始记录。
- 送礼记录及可推断的喜好层级。
- 事件 / 节日互动记录。
- NPC 当前状态：
  - 心情。
  - 注意度。
  - 开放度。
  - 当前回应倾向。
- NPC 对玩家的长期熟悉度。
- 每日互动节奏和关系舒适区。
- AI 提炼出的长期记忆：
  - 事实。
  - 偏好。
  - 承诺。
  - 边界。
  - 关系信息。
- 单独的玩家偏好记忆：
  - 喜欢的物品类别。
  - 讨厌的物品。
  - 生活习惯。
  - 价值观和态度。
  - 目标和计划。
- 单独的社区印象：
  - 其他 NPC 亲眼看到的玩家来往。
  - 亲近圈层里听来的低置信度消息。
  - 玩家和别的 NPC 关系变近的模糊印象。
- 单独的主动求助：
  - NPC 请求玩家找一件低价值物品。
  - 请求物品会根据季节、游戏阶段、角色倾向和关系阶段挑选。
  - 待完成 / 已完成 / 已拒绝 / 已过期状态。
- 单独的人际情绪和冲突记忆：
  - 当前情绪。
  - 情绪强度。
  - 冲突原因与严重度。
  - 恢复状态。
- 玩家专属称呼记忆。
- NPC 已执行过的 AI 世界动作。

每天开始时：

- 短期状态会向普通值回落。
- 人际情绪会向平静回落。
- 未解决冲突会按配置自然缓和。
- 当日计数会重置。
- 长期熟悉度和重要长期记忆不会像短期心情那样被清空。

#### 长期记忆检索与整理

长期记忆不会简单地全部塞进 AI 上下文。LivingNPCs 会先整理，再召回：

- 整理：按“类型 + 稳定主题”合并重复记忆，并保留更完整、重要度更高的表述。
- 强化：同一件事被多次提到时会提高 `TimesReinforced`，让它更像 NPC 真正在乎的长期印象。
- 标签：从 AI 返回的 `tags` 和记忆文本里推断场景标签，例如 `farming`、`mining`、`fishing`、`scholarly`、`flower`、`food`、`morning`、`night`。
- 保留分：按重要度、强化次数、记忆类型、最近更新时间和召回次数排序，最多保留每个 NPC 24 条长期记忆和 24 条玩家偏好记忆。
- 召回分：每次对话前，会根据当前地点、时间、最近互动、礼物、事件和求助计算相关性，只挑最相关的几条放进 ValleyTalk。
- 防复读：刚刚被召回过的记忆会短暂降权，避免 NPC 连续多次围着同一个旧话题打转。

调试摘要里会显示“当前检索长期记忆”和“当前检索玩家偏好”，方便确认这次到底是哪几条记忆进入了对话上下文。

### 情绪与冲突

当前长期情绪分为：

- `Happy`
- `Calm`
- `Jealous`
- `Worried`
- `Grateful`
- `Disappointed`
- `Uneasy`
- `Upset`
- `Angry`
- `Sad`

触发来源包括：

- 玩家对话里真实造成的情绪变化。
- 玩家道歉或修复关系。
- 送礼质量。
- 久别后再次见面，或 NPC 看到玩家明显受伤。
- 浪漫关系中的旁观情境。
- 玩家真正帮上大忙。
- 期待落空，例如答应的求助没有完成。

冲突不是普通尴尬，而是会继续影响关系的事件，例如明显冒犯、越界或严重欺骗等。LivingNPCs 会记录：

- 冲突原因。
- 严重度。
- 当前状态：`Active`、`Recovering`、`Resolved`
- 修复分数、道歉次数、解决时间。

当前恢复方式：

- AI 判断玩家确实道歉。
- 送出喜欢 / 最爱的礼物。
- 时间自然流逝。

LivingNPCs 还保存单独的 `关系信任`：

- `关系信任` 影响 NPC 愿意透露多深的秘密。
- 低信任时，即便原版心数不低，也会更少突然袒露很私人的内容。
- 高信任时，ValleyTalk 才会被允许在合适场景下写出更脆弱、更私人的坦白。

严重冲突会进入多步修复链，而不是靠一次友好回复直接清零：

- 需要真诚道歉。
- 需要一个合适的修复性礼物。
- 需要经过一定时间。
- 还需要一次真正谈开、能让 AI 标记为 `repairDelta` 的对话。

修复链会分阶段推进：

- 前期 NPC 仍然冷淡或保留距离。
- 条件逐步满足后，状态会从 `Active` 进入 `Recovering`。
- 真正修复后，关系会回暖；对非常严重但被认真修复的冲突，NPC 还会获得一点额外信任和熟悉度。

未解决冲突会影响行为和对话：

- NPC 更少靠近玩家。
- 更容易保留距离。
- 严重冲突下会阻止送礼、送钱、帮忙等亲近动作。
- ValleyTalk 会在生成对白时收到这层状态，因此 NPC 可以更冷淡，或在刚和解后自然说出“之前的事已经过去了”。

#### 情绪表达风格

同一个情绪状态不会再用同一种模板表达。LivingNPCs 会根据 NPC 的手写资料、Data/Characters 推断资料，以及少量关键角色专属规则，生成一层情绪表达风格：

- 直接型：不高兴会更容易说出来，边界更硬。
- 含蓄型：不高兴时更容易疏远、缩短回复，而不是解释一大段。
- 礼貌克制型：会保持礼貌，但明显降低热度。
- 温和型：仍然保留关心，但会让边界真实存在。
- 外露型：情绪更明显，善意回应后也更容易回暖。
- 反思型：更容易把不舒服说成观察、问题或需要时间消化的感受。

目前有专属处理的典型角色包括：

- `Penny`：柔软、克制，受伤时更安静，需要一点时间。
- `Shane`：嘴硬、防备，可能直接，但不轻易袒露脆弱。
- `Haley`：锋利、外露，小摩擦消得较快。
- `Harvey`：礼貌焦虑，更可能用迟疑和温和边界表达不舒服。

这层也会轻微影响实际系统数值：

- 情绪每天回落的速度。
- 冲突严重度每天自然衰减的速度。
- 道歉和修复对冲突的有效程度。
- 严重冲突至少需要经过的时间。

### 关系节奏

LivingNPCs 会同时看：

- 原版好感心数。
- LivingNPCs 长期熟悉度。
- 连续聊天天数。
- 今天第几次聊天。
- 距离上次聊天隔了多久。

据此区分：

- `Distant`
- `Familiar`
- `Friendly`
- `Trusted`
- `Intimate`

不同关系层级有不同的“每天聊几次还算自然”的舒适上限。这样 NPC 不会因为接入 AI 后就一直机械热情。

### ValleyTalk 集成

LivingNPCs 会把一份隐藏上下文推给 ValleyTalk，里面包含：

- 当前关系节奏。
- 当前场景。
- 当前世界阶段：
  - 社区中心 / Joja 路线。
  - 巴士、温室、矿车、姜岛、电影院等解锁。
  - 玩家职业倾向、农场规模、婚姻状态、孩子数。
  - 当前年份对应的“新来者 / 已融入 / 老住户”阶段。
- 当前 mood / inclination。
- 当前长期情绪与冲突。
- 最近送礼和事件。
- 长期记忆。
- 玩家偏好记忆。
- 社区印象：
  - 玩家最近和谁来往更频繁。
  - 谁亲眼看到过玩家帮忙。
  - 哪些事只是从亲近关系里听说。
- 对话驱动行为倾向。
- 当前是否还有待完成的主动求助。
- 玩家称呼偏好。
- 最近几条关键互动。

这份上下文只用于让 AI 更自然地延续关系，不应该被 NPC 原样说出来。

### 游戏进程感知

LivingNPCs 现在会把“这个存档已经走到哪里”压成一层独立的世界阶段摘要，而不只是看当天季节：

- `路线`：社区中心已修复、Joja 路线，或尚未决定。
- `设施解锁`：巴士、温室、矿车、姜岛、电影院。
- `玩家职业倾向`：根据真实 profession 记录归纳为耕种、钓鱼、采集、采矿、战斗。
- `农场规模`：根据当前作物、完工建筑和动物数量估计起步 / 小型运转 / 成熟 / 大型经营。
- `家庭阶段`：已婚对象与孩子数量。
- `时间阶段`：第一年春的新来者、第一年安顿中、第二年已融入、第三年及以后老住户。

这层信息会进入 ValleyTalk 的隐藏上下文，并附带明确约束：

- 第一年春天不能把玩家写成在镇上生活多年的老熟人。
- 第三年以后也不能还把玩家当成刚到镇上的新人。
- 已修好的设施会被当成日常事实，未解锁的设施不该被随口提成已经存在。
- 结婚、生子、农场做大以后，NPC 的话可以在合适时自然承认这些变化。

游戏进度现在又细分为两类：

- **公开事实**：镇上路线、公共设施解锁、年份阶段、婚姻和孩子。
- **私人可知信息**：农场大致规模、玩家职业倾向。

私人信息不会再无条件塞给每个 NPC。只有在这些情况下才会暴露：

- 关系已经足够熟。
- 当前 NPC 就在农场，能亲眼看到。
- 该 NPC 的职业或长期兴趣和这个领域有关，例如罗宾 / 玛妮对农场，威利对钓鱼，克林特对采矿。
- 玩家之前在和这个 NPC 的对话里亲口提到过，系统已经把它记进长期记忆。

#### SVE 专属进度感知

如果安装了 Stardew Valley Expanded，LivingNPCs 还会读取这个扩展自己的进度线索，把它们作为隐藏上下文交给 ValleyTalk：

- **SVE**：
  - 爷爷的棚屋是否已修复。
  - 是否已经认识 Apples。
  - 魔法林地 / Enchanted Grove 是否已确认。
  - Aurora Vineyard 是否已修复。
  - Crimson Badlands、Castle Village Outpost 是否已经去过。
  - 是否已接触 Susan。
  - Joja Emporium 是否已经去过。

这层不会把所有扩展信息无脑塞给每个人：

- SVE 角色会更自然地知道 SVE 进度。
- 原版 NPC 只会把扩展进度当作远景背景，除非当前地点、对话或长期记忆真的支持他们知道。
- 魔法线、冒险线、农场 / 葡萄园线、Joja 线等，会拿到更贴角色身份的提示。
- 未确认完成的扩展里程碑会被明确标成“未确认”，避免 NPC 提前说出玩家还没经历过的内容。

即便 NPC 知道农场已经变大，也只会拿到“大致规模”，而不是精确到作物格数、建筑数、动物数。这能让 NPC 更像在观察生活，而不是读取存档后台。

主动求助也会吃到这层进度：第一年春天只允许更轻、更短的一步请求；第一年后半段可以偶尔出现两步请求；等玩家已经真正安顿下来后，才允许更完整的多步骤委托。这样“新来镇上第三天就被托付一串复杂事情”的违和感会少很多。

世界进度现在还会经过 NPC 自己的立场再进入对白，而不是所有人只读同一张事实表：

- 社区型角色会把社区中心修复看成更有情感重量的事。
- `Morris` 会对社区中心修复和 Joja 路线给出和 `Pierre` 不同的反应。
- `Pam` 对巴士修复、`Willy` 对姜岛、农场型角色对温室、重视家庭的角色对结婚生子，都会拿到更贴近自己的态度提示。
- 这套态度层对原版和 SVE 角色都会生效；有手写档案的角色更准，没手写档案的角色会从 `Data/Characters` 与现有人物提示里保守推断。

### 调试与评估工具

现在可以用几种方式看清 LivingNPCs 为什么这样判断：

- `LeftShift + J`：查看附近 NPC 的状态摘要，输出到 SMAPI 控制台。
- `livingnpcs_debug [near|NPC名]`：输出某个 NPC 的状态、最近行为原因、求助生成适配、记忆召回。
- `livingnpcs_prompt [near|NPC名]`：输出即将注入 ValleyTalk 的完整隐藏上下文。
- `livingnpcs_export [near|all|NPC名]`：导出 Markdown 调试报告到 `Mods/LivingNPCs/debug_reports/<存档名>/`。
- `livingnpcs_eval`：运行一组轻量运行时诊断，确认关键人格化规则还在。
- `livingnpcs_giftmail`：诊断 LivingNPCs 礼物信——状态、邮箱位置、`Data/mail` 是否有对应条目、生成文本和孤儿死信。

报告里会显示：

- 每个 NPC 当前状态。
- 最近为什么选择了这个行为。
- 这次为什么适合 / 不适合提出求助。
- 哪些长期记忆、玩家偏好、社区印象被检索进 prompt。
- 完整 ValleyTalk 隐藏上下文预览。

仓库根目录还提供了离线回归检查：

```powershell
dotnet run --project LivingNPCs.Diagnostics\LivingNPCs.Diagnostics.csproj -- .
```

它不需要启动游戏，用来检查调试命令、情绪表达风格、求助解释、行为选择原因和文档说明没有被改丢。

### NPC 社会关系层

LivingNPCs 现在不再只记“这个 NPC 和玩家之间发生了什么”，还会保存一层小范围的社区印象。

当前会自然外溢的事实包括：

- 玩家和某个 NPC 最近来往得更频繁。
- 玩家完成了某个 NPC 的主动求助。
- 玩家在公开场合对某个 NPC 表现出明显亲密。

传播分两层：

- **亲眼看到**：同地图、距离足够近的 NPC 会记成较高置信度的 `Witnessed`。
- **亲近圈层听说**：系统会利用 `Data/Characters` 里的 `FriendsAndFamily` 关系，把消息记成较低置信度的 `CloseCircle`。
- **公共场所弱传播**：在 `Town` 和 `Saloon` 里，公开事实还会以更模糊的 `PublicRumor` 形式传给少量在场 NPC。

事实本身还有隐私级别：

- `Public`：例如最近来往更频繁，可在公共场所形成弱印象。
- `Personal`：例如帮忙、共同经历，只会在关系足够好的小圈子里继续扩散。
- `Private`：例如明显亲密，只会在直接目击者，或非常亲近且关系信任足够高的小圈子里流动。

此外，同一条社区印象进入不同 NPC 的上下文时，会再叠一层“口吻”：

- 温和角色更像关心。
- 含蓄角色更像试探。
- 直接角色更像一句干脆观察。
- 好奇角色更像追问。

因此：

- 海莉可能知道你最近常和艾米丽说话。
- 罗宾可能听说你帮过玛鲁。
- 在场的酒吧客人可能对你刚做过的事留下一个模糊印象。

这层信息不会被整座镇子无条件共享，也不会让 NPC 说出自己不该知道的细节。ValleyTalk 每次只会拿到一两条当前最相关的社区印象，并被明确要求：

- 亲眼所见可以更确定。
- 熟人转述和公共场所里的模糊消息要保留余地。
- 不要把八卦写成全知旁白。

#### 消息生命周期

社区消息现在不是永久常亮，而是会自己变旧：

- 刚发生时是 `fresh`，最容易进入下一次对话。
- 过几天后会变成 `settled`，还能提，但热度已经降下来了。
- 再往后会进入 `fading`，通常只作为很弱的背景，不再频繁被提。
- 过期或置信度太低后会被清掉。

不同来源衰减速度不同：

- 亲眼所见保留得更久。
- 熟人转述衰减更快。
- 公共场所里的弱印象最容易散掉。

#### 转述和失真

每天开始时，少量 NPC 可能会把自己还记得的新鲜消息继续讲给熟人圈：

- 爱表达、好奇的人更容易往外传。
- 含蓄、谨慎的人更容易把话留在自己心里。
- 私密消息不会继续多跳扩散。
- 个人消息通常只在亲近关系里走一小步。
- 公开消息才可能沿着更宽的圈层继续流动。

转述会记录层数：

- `0` 次：亲眼所见。
- `1` 次：第一手转述。
- `2+` 次：会越来越模糊。

系统会随着转述层数和失真度，把句子从“玩家帮了某人一个具体忙”逐渐收束成“听说玩家最近帮过某人一点忙”这类更含糊的说法，而不是让消息越传越精准。

#### 稳定社交圈

除了 `Data/Characters` 里的亲近关系，当前还内置了两类轻量稳定圈层：

- 酒吧常客圈。
- 年轻人圈。

这些圈层目前主要承载 **公开消息**，不会把私密事自动扩散出去。这样社区会有持续流动，但不会失去边界。

### AI 对话结果回写

ValleyTalk 当前会把以下隐藏元数据回传给 LivingNPCs：

- `rapportDelta`：本轮 AI 对话额外好感。
- `memories`：值得保留的长期记忆。
- `endConversation`：是否自然结束这轮交流。
- `ambientFollowUp`：稍后显示的头顶短句。
- `behaviorInfluences`：对话后几分钟到几天里的行为倾向。
- `emotionImpact`：这轮交流造成的人际情绪变化，以及是否构成道歉 / 修复。
- `conflicts`：需要长期记住的冲突事件。
- `helpRequests` / `helpRequestUpdates`：NPC 主动提出的小忙，以及后续完成 / 拒绝结果。
- `actions`：最多一个受控世界动作请求。

当前额外好感规则：

- AI 自己判断这轮聊得怎么样。
- 单次范围 `0-30`。
- 每个 NPC 每天总额外上限默认 `30`。
- 这是在原版每日对话好感之外追加的奖励。

### AI 世界动作

LivingNPCs 当前允许的世界动作只有：

- `give_small_gift`
- `give_meaningful_gift`
- `give_money`
- `companion_outing`
- `festival_interaction`
- `assist_quest`

但 AI 只能“申请”，不能直接执行。LivingNPCs 会重新检查：

- 关系是否足够。
- 当前位置是否合适。
- 今天是否已经执行过。
- 是否在冷却中。
- 是否超过配置上限。

补充说明：

- 如果对白里已经明确答应“现在一起去某处逛逛、看看风景或待一会儿”，LivingNPCs 会请求 `companion_outing`，而不是只停留在台词里。
- 如果 NPC 的对白是“等我一下，我准备一下再走”，系统会先记录一个很短的延迟，再开始出游，但不会额外显示准备完成气泡。
- 没有明确目的地的“一起走走”不会再触发移动动作。
- NPC 使用原版 `pathfindToNextScheduleLocation` 生成跨地图路线，在真实的门和出口处切图；不会再在玩家靠近出口时手动搬运到下一张地图。
- 到达后 NPC 会在语义锚点或安全评分选出的公共位置至少停留 60 游戏分钟，并保持稳定朝向，不再跟随玩家。
- NPC 安顿至少 20 游戏分钟、玩家仍在附近时，才可能低频显示一次与活动匹配的感叹号或爱心；整次出游最多一次，负面情绪或明显冲突时不会显示。
- 出游支持常见目标：农场、鹈鹕镇、山上、矿井、海边、煤矿森林、巴士站、酒吧、杂货店、图书馆 / 博物馆、诊所和常见 NPC 住宅。

当前主要规则：

- 小礼物：至少有一点熟悉；每天每个 NPC 最多一次 AI 礼物。
- 有意义礼物：至少友好，并且满足高关系 / 近期事件 / 重要长期记忆之一；默认 7 天冷却。
- 送钱：至少友好；金额默认 100g，受单次上限控制。
- 陪伴出游：至少有一点熟悉；只接受明确目的地，需能容纳路程和抵达后至少 60 游戏分钟，不改永久日程。
- 节日互动：只在事件 / 节日场景中允许。
- 任务协助：只围绕已有任务做轻量协助，不直接完成任务。

### 对话驱动行为

这层位于“即时世界动作”和“长期记忆”之间：

- 它不要求 NPC 当场立刻做事，也不会直接改写永久日程。
- ValleyTalk 会从对话里标注最多两条 `behaviorInfluences`。
- LivingNPCs 会把这些影响保存几天，并让后续被动行为、靠近 / 疏远倾向和地点反应参考它们。
- 例如被安慰后 NPC 会更愿意靠近；被冒犯后 NPC 会更可能后退；说想去海边后，如果后来真的在海边遇见，会自然接上一句。
- 当前仍保留安全边界：只有明确的 `companion_outing` 会临时让 NPC 跨地图移动，不会直接改写原版永久日程。

### 礼物选择

礼物分两层：

#### 小礼物

偏向日常、低价值，当前会结合：

- NPC 性格标签。
- 好感和熟悉度。
- 季节。
- 职业 / 背景。
- 当前对话内容。
- 长期记忆里出现的主题。
- 玩家偏好记忆；这部分权重高于普通主题记忆。

#### 有意义的礼物

偏向关系更深或特殊节点，只有在条件更强时才可能出现。它不是“小礼物升级版”，而是更少、更郑重的一层。

如果 NPC 已经记得玩家喜欢某类物品，礼物选择会明显向该类倾斜；如果记得玩家讨厌某类物品，相关候选会被压低。ValleyTalk 也会在合适时自然提一句类似“记得你喜欢这个”，让礼物和记忆接上，而不是凭空出现。

回礼与求助谢礼邮件会优先使用 NPC 性格模板。当前覆盖原版主要可社交 / 可送礼 NPC：所有原版可结婚对象，以及 Linus、Robin、Gus、Marnie、Willy、Evelyn、George、Pam、Clint、Wizard、Krobus、Pierre、Caroline、Demetrius、Jodi、Kent、Lewis、Sandy、Dwarf、Leo、Vincent、Jas；也覆盖 SVE 核心角色 Andy、Claire、Sophia、Susan、Victor、Olivia、Lance、Scarlett、Gunther、Martin、Morris（Magnus 复用 Wizard 模板）。每人每类邮件各有 3 个稳定变体，未覆盖 NPC 会回退到通用模板。生日当天送给 NPC 的礼物会使用单独的生日谢礼邮件，按 NPC 性格组选择 3 个稳定变体之一，并在次日寄出。

### 主动求助

LivingNPCs 会保存一层“主动求助”，让 NPC 自己开口请玩家帮一个小忙。

当前支持：

- `item_request`：让玩家带来一个低价值、白名单内的物品。
- 新求助现在只接受物品请求；请教问题会保留为普通 AI 对话，不再进入任务栏。

默认触发条件：

- 同一个 NPC 当前没有别的待完成求助。
- 关系信任达到阈值，且已经有一点熟悉度。
- 没有明显未解决冲突。
- 当前不是明显生气 / 不悦。
- 最近几天没有刚发起过新的求助。

求助有两种出现时机：一是当前对话刚好聊到合适的点；二是每天与某个 NPC 第一次对话时，按配置概率（默认 `25%`）单独掷一次，决定该 NPC 是否主动在这次聊天里开口求助——即便玩家没把话题引过去。两种方式都仍要先满足上面的条件。

当前完成方式：

- 新求助会先进入 `Offered` 状态：NPC 只是开口问了，玩家还没有答应。
- 玩家在后续 AI 对话里明确答应后，ValleyTalk 回传 `accepted`，求助才进入 `Pending` 并显示到原版任务栏。
- 玩家婉拒时，ValleyTalk 回传 `declined`，这次求助不会变成任务，只留下轻量关系记忆。
- 物品求助：玩家把当前步骤需要的物品直接送给该 NPC 后推进或完成。
- 多步骤求助最多 3 步；每一步都必须是当前阶段允许的物品请求。

当前查看方式：

- LivingNPCs 会把待完成求助直接投影进原版任务日志。
- 任务条目会显示：
  - 谁向你求助。
  - 需要什么物品。
  - 还有多久到期。
- 任务条目不再显示奖励提示，也不显示可领取的奖励框（与原版“物品配送”委托一致）。
- 底层仍由 LivingNPCs 自己维护，不会把这套系统硬塞成原版公告栏任务。

生成请求时，ValleyTalk 会额外看到一条 `Help-request fit` 提示，要求它尽量结合：

- 角色职业。
- 角色性格。
- 当前季节。
- 这轮对话里刚刚聊到的内容。

因此它不应该让每个 NPC 都机械地问同一种小忙。

物品请求还会受游戏进度约束：

- 只从当前季节能自然获得的低价值季节物品里挑。
- 矿井未开放时，不会向玩家要石英。
- 玩家还没有深入矿井时，不会向玩家要紫水晶。
- 关系较低时只会挑非常轻的当季采集物或花。
- 关系提高后，才会加入面包、咖啡、饼干这类更日常、更有人情味的低价值物品。
- 如果此刻没有合适物品，系统会让 NPC 暂时不开求助任务，而不是改成问题型求助。

完成后：

- NPC 会进入轻度 `Grateful`。
- 关系信任和熟悉度都会提高。
- 完成多步骤求助的中间步骤时，会先提高一点信任和开放度；完整好感奖励会等最后一步完成。
- 额外增加 `50-100` 点好感，实际范围可在配置里改。
- 若该求助带有金币奖励，会在交付当下立即发放（与原版“物品配送”委托一致），不再延到次日来信。
- 若允许 NPC 小礼物，完成后会安排次日寄来一份小谢礼，不会当场塞进背包。
- 隔天可能用头顶气泡自然补一句感谢。
- 这次帮助会记成一条“共同经历”里程碑，供后续对话自然回忆。
- 若这次求助被 AI 标记为 `deeper_relationship`，完成后的共同经历会更容易在后续对话里推动关系加深。

若玩家明确拒绝，或请求到期仍未完成：

- 如果玩家从未答应，只是没有回应，反应会很轻。
- 如果玩家已经答应却没完成，NPC 会更失望，并按角色性格表现出不同反应：有的人会直接，有的人会退缩，有的人仍保持温和。
- 普通未完成求助不会直接变成高强度冲突。

这一层已经接进原版任务栏；不过底层仍然是 LivingNPCs 自己维护的轻任务系统，而不是直接复用原版公告栏任务逻辑。

### 自定义 NPC 兼容

- 原版角色有内置倾向。
- SVE 当前内置识别到的角色已经补齐手写摘要，并处理了常见内部别名。
- 对未手写配置的 NPC，会从 `Data/Characters` 推断基础性格。
- 已对 SVE 做了专门识别：
  - 有些角色有手写摘要。
  - 其余角色会保留所属 Mod 的世界观提示，并回退到基础推断。
- `npc_profiles/` 目录支持社区直接放入 JSON 人物资料：
  - 可新增自定义 NPC；
  - 也可覆盖已有条目做修正；
  - 模板与提交流程见 [`npc_profiles/README.md`](./npc_profiles/README.md)。

## 使用方法

默认快捷键：

- `LeftShift + H`：触发附近 NPC 的一次小行为。
- `LeftShift + J`：查看附近 NPC 的状态和记忆。

Generic Mod Config Menu 中可以调：

- 行为快捷键。
- 记忆数量。
- AI 对话额外好感。
- 对话驱动行为开关、行为影响最长保留天数。
- 主动求助开关、待完成上限、冷却天数、最低关系信任、每日主动开口概率、完成奖励范围。
- 世界动作开关。
- 礼物、送钱、陪伴出游的限制。
- 每日情绪衰减、每日冲突衰减。
- 被动行为概率。
- ValleyTalk 桥接。
- 精简版隐藏上下文（`ConcisePromptContext`，默认关闭）。
- AI 行为规划器。

## AI 行为规划器

这是一个与 ValleyTalk 对话生成分开的可选功能，默认关闭。

启用后，LivingNPCs 可以向 OpenAI-compatible 接口请求一个行为意图：

```json
{
  "intent": "FacePlayer",
  "reason": "short in-world reason",
  "emoteId": 16
}
```

但即使模型返回了内容，LivingNPCs 仍会做本地安全校验；失败时会自动回退到规则行为，不会因为接口异常卡住游戏。
